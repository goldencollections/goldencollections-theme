## Executive Takeaway

This is not currently a mass clean-product indexing emergency.

The first 1,000 GSC examples are mostly duplicate crawl-noise URLs generated by Shopify recommendation, variant, currency, search, sitemap, feed, and tracking patterns. Only 26 of the 1,000 examples are clean product URLs with no query string.

The best next implementation sprint is not mass noindexing and not bulk redirects. It is a careful source-link cleanup to reduce how often crawlable internal links expose `pr_*`, `variant`, `currency`, `_pos`, `_psq`, `_ss`, and similar query URLs.

## What Not To Do

- Do not validate the entire `Crawled - currently not indexed` issue in GSC yet.
- Do not add blanket `noindex` to products, collections, variants, or sitemap/AI files.
- Do not bulk redirect parameterized product URLs. They already canonicalize and are often valid user-facing URLs.
- Do not remove valid variant behavior unless the product UX and Shopify cart behavior are checked.
- Do not block all query URLs in robots.txt as a first move; that can prevent Google from seeing canonical tags on URLs it already discovers.

## Recommended Next Sprint

### 1. Parameterized Internal Link Cleanup

Dry-run first:

- Find every theme source that renders product links from recommendations, related products, recently viewed, search results, predictive search, product cards, and collection grids.
- Identify which sources emit URLs with `pr_prod_strat`, `pr_rec_id`, `pr_rec_pid`, `pr_ref_pid`, `pr_seq`, `variant`, `currency`, `_pos`, `_psq`, `_ss`, `_v`, `country`, or `utm_*`.
- For product-card anchors where the query string is not required for customer function, change the rendered link to the clean product URL.
- Preserve any link where the query string is needed for a real selected variant, selling plan, cart behavior, campaign landing page, or analytics requirement.

Implementation should be scoped and verified with rendered product page, collection page, related-products, and search-result checks.

### 2. Clean Product Indexing Review

Review the 26 clean product URLs in the sample:

- Current status and canonical.
- Product availability and inventory.
- Product title quality and duplication.
- Main image and alt text quality.
- Description uniqueness and buyer usefulness.
- Product schema completeness.
- Internal links from relevant collections.
- Whether the product belongs in sitemap and collection navigation.

This is the part most likely to create real incremental organic traffic.

### 3. Legacy Redirect Review

Review odd clean/legacy redirects found in the sample, especially when the old path and new target differ semantically:

- `goddess-lakshmi-doll-face` -> `lord-vishnu-face-alangaram-tanjore-idol`
- `fashion-accessories/mens-bracelets` -> `bharatanatyam-earrings-collection`

If these are intentional historical redirects, leave them. If not, remap to a closer product/collection or return a cleaner 404 only when no suitable target exists.

### 4. Restore Full API Export

Reauthorize the Search Console API token so the full 9,873 URL set can be exported and diffed over time. The UI only exposes 1,000 example rows, which is enough for pattern analysis but not enough for exact cleanup accounting.
